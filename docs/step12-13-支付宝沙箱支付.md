# 第12+13步：支付宝沙箱支付（后端+前端）

## 目标

实现支付宝沙箱支付功能，包括VIP套餐展示、支付订单创建（RSA2签名）、支付宝异步通知处理、支付状态查询、VIP续期逻辑。

## 核心产出

| 文件 | 作用 |
|------|------|
| `config/AlipayConfig.java` | 支付宝配置属性类（APPID、密钥、网关、回调URL等） |
| `mapper/PaymentRecordMapper.java` | 支付记录数据访问 |
| `dto/VipProductVO.java` | VIP套餐返回VO |
| `service/PaymentService.java` | 支付服务接口 |
| `service/impl/PaymentServiceImpl.java` | 支付服务实现（订单创建、RSA2签名、GET重定向、回调处理、模拟支付） |
| `controller/PaymentController.java` | 支付API控制器（创建/通知/返回/状态/模拟支付） |
| `api/payment.ts` | 支付API调用封装 |
| `views/Payment.vue` | 充值页面（三档套餐卡片、支付交互） |
| `views/PaymentResult.vue` | 支付结果页（自动调用模拟支付） |

## API接口

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/payment/products` | 无需 | 获取VIP套餐列表 |
| POST | `/api/payment/create` | 需要 | 创建支付订单 → 返回GET跳转URL |
| POST | `/api/payment/notify` | 无需 | 支付宝异步通知回调（需外网穿透） |
| POST | `/api/payment/mock/{orderNo}` | 需要 | 模拟支付成功（开发测试用） |
| GET | `/api/payment/status/{orderNo}` | 需要 | 查询订单状态 |
| GET | `/api/payment/return` | 无需 | 支付宝同步跳转（重定向到前端结果页） |

## 关键功能

- **三档套餐**: 月度VIP(30元)、季度VIP(80元「最划算」)、年度VIP(288元「省更多」)
- **RSA2签名**: 后端使用 Java `Signature` 类实现 SHA256withRSA 签名，参数按字母排序后签名
- **GET重定向（防乱码）**: 使用 `window.location.href = payUrl` 直接跳转GET链接，所有参数由后端 `URLEncoder.encode(value, "UTF-8")` 编码，彻底避免浏览器 POST 时 GBK 编码导致的中文乱码问题
- **签名降级**: 如果 RSA2 签名失败（密钥格式不对等），自动降级为模拟支付（mockPaySuccess）
- **幂等处理**: 订单状态已为 SUCCESS 时跳过重复处理（防止支付宝重复通知）
- **VIP续期**: 如果用户已是VIP且未过期，新购买的套餐时间累加
- **JWT Token自动刷新**: 支付成功后后端返回含VIP角色的新JWT Token，前端替换旧Token后自动更新用户信息，无需重新登录
- **管理员保护**: 管理员(ADMIN)角色在支付时不会被覆盖为VIP

## 核心实现

### 支付流程
```
前端选套餐 → POST /api/payment/create → 后端生成订单 + RSA2签名
    → 返回 GET 跳转URL（含所有参数） → window.location.href 跳转
    → 支付宝沙箱收银台 → 用户扫码/账号登录支付
    → 异步通知 POST /api/payment/notify（外网穿透） → 更新VIP状态
    → 同步跳转 GET /api/payment/return → 前端结果页
```

### 签名实现
```java
// 参数按字母排序后拼接
String signContent = params.entrySet().stream()
    .sorted(Map.Entry.comparingByKey())
    .map(e -> e.getKey() + "=" + e.getValue())
    .collect(Collectors.joining("&"));
// SHA256withRSA 签名
Signature signature = Signature.getInstance("SHA256withRSA");
signature.initSign(privateKey);
signature.update(signContent.getBytes(StandardCharsets.UTF_8));
String sign = Base64.getEncoder().encodeToString(signature.sign());
```

### 内网穿透
支付宝异步通知需要外网可访问的地址，开发环境支持两种方案：
- **花生壳**: 更稳定，免费隧道可用
- **NATAPP**: 免费隧道但稳定性差，提供 `natapp.exe` 自动保活脚本

## 注意事项

- 需要在 `application.yml` 中替换为真实的沙箱APPID、应用私钥（PKCS8格式）、支付宝公钥
- `notify-url` 必须为外网可访问的地址（花生壳或NATAPP），否则支付宝无法回调
- 沙箱网关地址：`https://openapi-sandbox.dl.alipaydev.com/gateway.do`
- 注意区分 **应用公钥**（上传到沙箱页面）和 **支付宝公钥**（填到配置文件中），两者不同
