# 第14+15步：关系图谱（Neo4j + 前端展示）

## 目标

使用 Neo4j 图数据库存储电影、演员、导演、分类之间的关系数据，实现可视化关系图谱展示，支持搜索查看特定电影或人物的关系网。

## 核心产出

### Neo4j 数据导入
| 文件 | 作用 |
|------|------|
| `import_neo4j.py` | 从MySQL批量导入数据到Neo4j（Person/Movie/Category节点及关系） |
| `config/Neo4jConfig.java` | Neo4j驱动配置 Bean |

### 后端API
| 文件 | 作用 |
|------|------|
| `controller/GraphController.java` | 图谱API控制器 |
| `service/GraphService.java` | 图谱服务接口 |
| `service/impl/GraphServiceImpl.java` | 图谱查询实现（Cypher查询） |

### 前端页面
| 文件 | 作用 |
|------|------|
| `views/Graph.vue` | 关系图谱页面（vis-network力导向图） |
| `api/graph.ts` | 图谱API调用封装 |

### 其他
| 文件 | 作用 |
|------|------|
| `public/vis.min.js` | vis-network图库（本地部署） |

## API接口

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/graph/hot?limit=20` | 无需 | 获取热门电影关系图数据 |
| GET | `/api/graph/search?keyword=&limit=30` | 无需 | 搜索电影或人物关系图 |

## Neo4j 数据模型

```
节点类型：
  - Person {id, name, type: "actor"|"director"|"both"}
  - Movie {id, title}
  - Category {id, name}

关系类型：
  - (Person)-[:ACTED_IN]->(Movie)      演员 → 电影
  - (Person)-[:DIRECTED]->(Movie)      导演 → 电影
  - (Movie)-[:BELONGS_TO]->(Category)  电影 → 分类
```

## 关键功能

- **默认展示TOP20热门电影**关系网（含演员、导演、分类）
- **搜索电影名/演员名/导演名**，只显示局部关系网
- **五类节点区分**：
  - 🔴 红色圆形 = 电影
  - 🔵 蓝色圆形 = 只当演员
  - 🟢 绿色方形 = 只当导演
  - 🟣 紫色菱形 = 既是演员又是导演
  - 🟡 黄色菱形 = 分类
- **自导自演双线**：既是演员又是导演的人拉两条线（🔵ACTED_IN + 🟢DIRECTED）
- **力导向图布局**：节点拖拽、缩放、悬停查看信息

## 注意事项

- 需要先安装并启动 Neo4j（版本 5.x）
- Neo4j 默认账号密码为 `neo4j`，需先修改密码
- 运行 `python import_neo4j.py` 将MySQL数据导入Neo4j（已预导入）
- vis-network 库已本地部署至 `public/vis.min.js`，无需CDN加载
- `/api/graph/**` 已在 WebMvcConfig 中配置放行
