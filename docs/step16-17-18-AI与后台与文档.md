# 第16步：Spring AI 整合（续）

## 目标
实现 AI 智能推荐和 AI 智能搜索功能。当用户有观看历史时，基于历史推荐同类型电影；无历史时推荐热门电影。

## 核心产出
| 文件 | 作用 |
|------|------|
| `controller/AIController.java` | AI 推荐 + AI 搜索接口 |
| `api/ai.ts` | AI API 前端调用封装 |

## API接口
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/ai/recommend` | AI智能推荐（需认证，基于观看历史） |
| GET | `/api/ai/search?q=` | AI智能搜索 |

## 关键说明
- 推荐逻辑：查用户观看历史 → 找同分类未看过电影 → 不够则补充热门
- Spring AI OpenAI 配置已预留在 application.yml 中，可通过环境变量 `AI_API_KEY` 激活

# 第17步：管理后台

## 目标
实现简单的管理后台，包括管理员登录后的 Dashboard 数据概览、电影管理、用户管理功能。

## 核心产出
| 文件 | 作用 |
|------|------|
| `views/Admin.vue` | 管理后台布局（侧边栏+路由视图） |
| `views/admin/Dashboard.vue` | Dashboard 数据概览（含ECharts图表） |
| `views/admin/MovieManage.vue` | 电影管理（列表/状态修改） |
| `views/admin/UserManage.vue` | 用户管理（列表/角色/状态修改） |

## 路由结构
管理后台使用 Vue Router 嵌套路由，访问 `/admin/*` 需要 ADMIN 角色：
- `/admin` → 自动重定向到 `/admin/dashboard`
- `/admin/dashboard` → 数据概览（电影数/用户数/评论数 + 每日播放趋势折线图 + 用户构成饼图 + TOP10播放量条形图）
- `/admin/movies` → 电影管理（上下架、VIP设置）
- `/admin/users` → 用户管理（启用/禁用、角色修改）

## Dashboard 图表
- **每日播放趋势**: ECharts 柱状图，显示最近7天播放量变化，渐变色柱体
- **用户构成**: 环形饼图，展示VIP会员与普通用户占比
- **播放量TOP10**: 横向条形图，10种不同颜色区分排名，展示播放量最高的电影

## 说明
- 路由守卫在 `router/index.ts` 中配置，`/admin/*` 路径需要 token 且 role=ADMIN
- 管理后台不验证非管理员访问，依赖后端的权限拦截器
- 实际操作通过调用后端 API 完成（更新电影状态、用户状态等）
